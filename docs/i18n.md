# Internationalisation

The bot ships English and Italian translations and supports user-level language switching
through `/lang`. Adding a third language is a self-contained `.po` file plus a PR.

## Architecture

- Library: stdlib `gettext` + `babel` for extraction.
- Source language (`msgid`): English. We translate from English into other locales.
- Locale tree: `src/parcel_tracker/i18n/locale/<lang>/LC_MESSAGES/messages.po`.
- Compiled `.mo` files are produced at build time (Docker stage) — they are **not** committed.
- Per-user persistence: `users.language` column. Default = `DEFAULT_LANGUAGE` env (default `en`).
- Telegram `language_code` from updates is **not** used by default for privacy. A user must
  opt in with `/lang it`.

## Switching language at runtime

In Telegram:

```
/lang en    # English (default)
/lang it    # Italiano
/lang       # show current language and available locales
```

The bot stores the choice in the SQLite DB and uses it for every future reply.

## Adding a new language

Example: French (`fr`).

1. Copy the `en` locale as a starting point:

   ```bash
   mkdir -p src/parcel_tracker/i18n/locale/fr/LC_MESSAGES/
   cp src/parcel_tracker/i18n/locale/en/LC_MESSAGES/messages.po \
      src/parcel_tracker/i18n/locale/fr/LC_MESSAGES/messages.po
   ```

2. Open the new file. Update the metadata block at the top:

   ```
   "Language: fr\n"
   "Plural-Forms: nplurals=2; plural=(n > 1);\n"
   ```

3. Translate every `msgstr ""` line. Keep `msgid` lines untouched. Mind the placeholders —
   strings with `{tracking_number}` must keep that placeholder.

4. Compile and run locally:

   ```bash
   python -m parcel_tracker.i18n.build
   pytest tests/unit/i18n/
   pytest tests/integration/test_i18n_flow.py -k "test_lang_switches_to_fr" -v
   ```

5. Open a PR with the new `.po` file. CI will compile the `.mo` and run the suite.

## Re-extracting strings after code changes

If you add a new user-facing string in the bot code, regenerate the `.pot` template:

```bash
pybabel extract -F babel.cfg -o src/parcel_tracker/i18n/locale/messages.pot src/parcel_tracker/
pybabel update  -i src/parcel_tracker/i18n/locale/messages.pot \
                -d src/parcel_tracker/i18n/locale -D messages
```

This adds new `msgid` entries to every locale `.po` with empty `msgstr`. Translators fill them
in; CI will not block on missing translations (untranslated strings fall back to the English
source).

## Plurals

Use `ngettext` (already imported as `_n`) for any string that can be plural:

```python
from parcel_tracker.i18n import _n

await update.message.reply_text(
    _n(
        "{count} parcel updated",
        "{count} parcels updated",
        n=count,
    ).format(count=count),
)
```

## Style guide for translators

- Keep emoji in place (they convey status colour at a glance).
- Keep HTML tags (`<b>`, `<code>`) verbatim — the Telegram client renders them.
- Keep placeholders like `{tracking_number}` verbatim — Python `str.format()` consumes them.
- Match the politeness register of the source: the English baseline uses second-person
  imperative for commands ("Add a parcel", not "Please add a parcel"). Translate accordingly.
